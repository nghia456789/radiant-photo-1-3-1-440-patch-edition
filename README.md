## Command-Line Integration (Console Invocation)

For power users and script-based workflows, Radiant Photo supports CLI invocation. Here's an example:

```bash
# Batch process all .CR3 files in a directory
radiant-photo --input ./raw_shoot/ \
              --output ./exports/ \
              --profile portraits_2026.rpf \
              --format TIFF \
              --compression LZW \
              --color-space AdobeRGB \
              --batch-mode sequential \
              --threads 8 \
              --dry-run  # Remove to execute
```

### Arguments Explained

| Flag | Type | Description |
|------|------|-------------|
| `--input` | Path | Source folder (recursive) |
| `--output` | Path | Destination folder |
| `--profile` | String | Profile filename (must be in Profiles/) |
| `--format` | String | `TIFF`, `JPEG`, `PNG`, `PSD`, or `DNG` |
| `--compression` | String | Only for TIFF: `NONE`, `LZW`, `DEFLATE` |
| `--color-space` | String | `sRGB`, `AdobeRGB`, `ProPhoto` |
| `--threads` | Integer | CPU threads (max: logical cores - 2) |
| `--dry-run` | Flag | Simulate without writing files |

**Use case:** A wedding photographer can trigger a batch export from a shell script after an all-night shoot, returning to finished TIFF files in the morning.
